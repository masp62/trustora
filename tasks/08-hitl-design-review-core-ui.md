# #8 — HITL: Design Review — Core UI (Post Cards, Detail, Nav, Profile Layout)

**Type:** HITL
**Blocked by:** #7 — Post Detail Page & Slug URL

## What to build

Design review checkpoint. Review and approve the core UI components and layouts that all remaining slices will build on: the post card, the post detail page, the photo gallery, the global navigation shell, and the user profile page layout.

Aspects to evaluate:

- **Post card:** lead photo, title, location chip, author avatar + name, like count — visual hierarchy, hover states, truncation behavior
- **Post detail page:** full-width photo gallery placement, story body typography, metadata chips (tags, trip type, location), author block, transition between gallery and body
- **Photo gallery:** navigation controls (arrows, dots), keyboard and touch support, aspect ratio handling for mixed-size photos
- **Navigation shell:** header (logo, nav links, auth state), mobile hamburger/drawer, active route indication
- **Profile page layout:** avatar size, bio placement, post grid, follower/following count placement, follow button position
- **Design language validation:** warm color palette, whitespace, photography-forward feel consistent across components

## Acceptance criteria

- [x] Post card design reviewed and approved (or change requests issued)
- [x] Post detail page layout reviewed and approved (or change requests issued)
- [x] Photo gallery behavior reviewed and approved (or change requests issued)
- [x] Navigation shell reviewed and approved (or change requests issued)
- [x] Profile page layout reviewed and approved (or change requests issued)
- [x] All design change requests from this review resolved before feeds and profiles are implemented (#13, #15)
